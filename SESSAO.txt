   - Faz requisição HTTP via axios à página do produto
   - Extrai nome via og:title (fallback para <title>)

   - Extrai preço com patterns específicos por plataforma (Amazon, Mercado Livre, Shopee)
   - Fallback automático para dados do texto quando o fetch falha (404, timeout, bloqueio)
   - User-Agent real de navegador (Chrome 125) para evitar bloqueios
   - Timeout configurável de 10s
   - Tratamento de erros: 404, timeout, rede — todos com graceful handling
   - Exportado via barrel de utils (src/utils/index.ts)

3. Integração Product Fetcher + CLI
   - Sempre que uma URL é detectada, o CLI dispara fetch automático
   - Preview enriquecido: mostra nome + descrição + imagem vindos do link
   - Dados do link identificados com tag "(via link)" no preview
   - Dados do texto usados como fallback quando fetch falha ou não retorna dados

2. Diagnóstico das falhas
   - Shopee: URLs no formato curto `s.shopee.com.br/XXXXX` e `shp.ee/XXXXX` não eram reconhecidas
   - Shopee: URLs com formato `/{texto}-i.{shopId}.{itemId}` (ex: `/opaanlp-i.12345.67890`) não extraíam product data
   - Shopee: query params no final da URL (`?__mobile__=1&...`) quebravam o regex `/\/(\d+)\/(\d+)$/`
   - AliExpress: URL curta `s.click.aliexpress.com/e/_XXXXX` resolvia para página HTML, não permitindo extrair productId direto
   - Cupom detector: regex `/\bcupom\b/i` não detectava "RENOVADOS CUPONS SHOPEE" (plural)
   - tidd.ly: URLs do tidd.ly não eram reconhecidas (domínio não mapeado)

3. Fixes implementados

   a) parser.ts — Novos padrões Shopee:
      - `/https?:\/\/shp\.ee\/[a-zA-Z0-9]+/g` (URLs curtas)
      - `/https?:\/\/s\.shopee\.com\.br\/[a-zA-Z0-9]+/g` (URLs curtas .br)
      - `/-i\.(\d+)\.(\d+)/g` (formato /{texto}-i.{shopId}.{itemId})
      - `extractLinksFromMessage` agora retorna TODAS as URLs (removeu filtro `isProductLink`)

   b) shopee.ts — Resolução de URL + novos padrões de extração:
      - `resolveShortUrl()` — segue HTTP redirects (axios com maxRedirects: 0, intercepta 302)
      - Extração de shopId/itemId do padrão `-i.{shopId}.{itemId}` no extractProductData
      - Regex `/\/(?:[^/]+\/)?(\d+)\/(\d+)(?:\?|&|$|[/#])/i` — tolera query params no final

   c) aliexpress.ts — Resolução de URL curta:
      - `resolveAliExpressShortUrl()` — segue redirect de `s.click.aliexpress.com/e/_XXXXX`
      - Extrai productId da resolved URL (formato `/item/{id}.html`)

   d) coupons/detector.ts — Plural:
      - `/\\bcupons?\\b/i` — aceita "cupons" (plural) além de "cupom"

   e) monitor.ts — URL desconhecida + fallback _data:
      - `resolveUnknownUrl()` — segue redirect de domínios não reconhecidos, re-detects platform
      - Extração de `fullText` do `message.body` com fallbacks para `message._data` (canonicalUrl, matchedText, url, link, title, description, body)
      - Debug detalhado quando nenhum link é encontrado (bodyLength, lastPart, bodyCompleto)
